3. docker run --name test-memoria --memory=100m --memory-swap=100m busybox sh -c 'i=0; a=x; while true; do a=$a$a; i=$((i+1)); echo Iter $i; done' -> Si hacemos docker ps -a vemos como se cerró con un exit(137), por lo que lo mató el sistema porque no le quedaban recursos.

4. docker run --rm --cpus=0.5 busybox sh -c "while true; do :; done"  -> Si te metes en otra terminal con docker stats podemos ver como consume toda la cpu que le hemos dado.

Comandos:

Docker stats ---> Muestra el uso de recursos de los contenedores en tiempo real

Docker inspect <nombre_contenedor> | grep -i Memory ---> Muestra la cantidad de memoria asignada al contenedor

Docker inspect <nombre_contenedor> | grep -i NanoCpus ---> Muestra la cantidad de CPU asignada al contenedor
